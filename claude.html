Section 1

Claude & Claude Cowork Best Practices

Claude is our primary AI workhorse. Use Chat for thinking and Cowork for doing — and set up each research stream once so you never pay to re-explain context.

Chat vs Cowork: Pick the Right Mode

Claude Chat

For thinking, drafting, and analysis

  • Reasoning through a research question or analytical framework
  • Drafting and editing report sections, summaries, briefings
  • Reviewing translations from Burmese, Thai, Shan, and other languages
  • Interpreting patterns in data you describe or paste as excerpts
  • Structuring interview notes or field observations (anonymized)
Claude Cowork

For working directly on files

  • Building deliverables: Word reports, Excel workbooks, slide decks
  • Cleaning, restructuring, and converting data files
  • Multi-step tasks: "process these 12 CSVs and produce a summary table"
  • Repetitive document work across many files in a project folder
  • Writing and testing scripts (Python, SQL) against sample files

Rule of thumb: if the answer belongs in a file, start in Cowork. If the answer belongs in your head, start in Chat. Pasting file contents into Chat wastes tokens twice — once to paste, once when Claude repeats it back.

Set Up One Project per Research Stream

Claude Projects hold persistent context so you stop paying to re-explain it in every conversation.

Create a Project for each long-running engagement or workstream — for example: a monitoring-and-analysis engagement, a supply-chain risk study, SAM product development, or a mapping project. In each Project's knowledge/instructions, store the stable context once:

  • The engagement's purpose, scope, and client-approved terminology (anonymized to the level the tier rules allow)
  • Our methodology notes — e.g., how we verify sources, our scoring rubric, our visualization standards
  • Output templates: report structure, citation style, standard caveats
  • Schema descriptions for relevant datasets (structure only, no data rows)
Good — context lives in the Project

Project instructions contain the report template and methodology. Each new chat: "Draft the Q2 trend section using the attached excerpt." ~200 tokens of prompt.

Wasteful — context re-pasted every time

Every chat opens with 3,000 words of background, the full template, and last quarter's report pasted in before the actual request.

Cowork Folder Discipline

Cowork works on folders you explicitly connect. Treat that connection like handing over a filing cabinet:

  • One task folder per engagement. Create a dedicated working folder containing only the files the task needs. Connect that — never a parent folder, your Desktop, or your whole Drive sync folder.
  • No client deliverable folders. Final confidential deliverables live in their own protected location that is never connected to Cowork. Work on drafts in the task folder; move finals out when done.
  • Strip before you connect. Before connecting a folder, scan it: any file naming sources, field staff, or persons of interest that the task doesn't require gets moved out first.
  • Review what Cowork produced before moving files onward. You are the quality gate, exactly as with a junior analyst's work.

Connector Rules

ConnectorStatusWhy
Email (Gmail / Outlook)ProhibitedEmail holds client correspondence, contracts, and source contacts. No AI tool connects to ENVISAGE email, ever.

Prompting Patterns for Research Work

Give the role, the source, the output shape — once

You are assisting an ENVISAGE analyst on a supply-chain study.
Source: the excerpt below (public news reporting, translated).
Task: extract company names, locations, and alleged practice
into a 3-column table. Flag low-confidence extractions.

Make Claude ask before generating long output

For big deliverables, add: "Before drafting, list your assumptions and an outline in under 150 words and wait for my confirmation." A 100-token outline correction is far cheaper than regenerating a 3,000-token draft.

Iterate on sections, not documents

Ask for revisions to the specific section that needs work ("revise paragraph 2 only — keep everything else"), not a full regeneration.

แนวปฏิบัติที่ดีสำหรับ Claude และ Claude Cowork

Claude คือเครื่องมือ AI หลักของเรา ใช้ Chat สำหรับการคิด และ Cowork สำหรับการลงมือทำ — ตั้งค่าแต่ละสายงานวิจัยเพียงครั้งเดียว เพื่อไม่ต้องเสียค่าอธิบายบริบทซ้ำ

Chat กับ Cowork: เลือกโหมดให้ถูก

Claude Chat

สำหรับการคิด ร่าง และวิเคราะห์

  • คิดวิเคราะห์โจทย์วิจัยหรือกรอบการวิเคราะห์
  • ร่างและแก้ไขส่วนต่าง ๆ ของรายงาน บทสรุป เอกสารบรรยายสรุป
  • ตรวจคำแปลจากภาษาพม่า ไทย ไทใหญ่ และภาษาอื่น
  • ตีความรูปแบบในข้อมูลที่คุณอธิบายหรือวางเป็นข้อความบางส่วน
  • จัดโครงสร้างบันทึกสัมภาษณ์หรือข้อสังเกตภาคสนาม (ปกปิดตัวตนแล้ว)
Claude Cowork

สำหรับทำงานกับไฟล์โดยตรง

  • สร้างงานส่งมอบ: รายงาน Word, ไฟล์ Excel, สไลด์นำเสนอ
  • ทำความสะอาด ปรับโครงสร้าง และแปลงไฟล์ข้อมูล
  • งานหลายขั้นตอน: "ประมวลผล CSV 12 ไฟล์นี้แล้วสร้างตารางสรุป"
  • งานเอกสารซ้ำ ๆ กับหลายไฟล์ในโฟลเดอร์โครงการ
  • เขียนและทดสอบสคริปต์ (Python, SQL) กับไฟล์ตัวอย่าง

หลักง่าย ๆ: ถ้าคำตอบควรอยู่ในไฟล์ เริ่มที่ Cowork ถ้าคำตอบควรอยู่ในหัวคุณ เริ่มที่ Chat การวางเนื้อหาไฟล์ลงใน Chat เปลืองโทเคนสองต่อ — ตอนวาง และตอนที่ Claude พิมพ์ซ้ำกลับมา

ตั้งหนึ่ง Project ต่อหนึ่งสายงานวิจัย

Claude Projects เก็บบริบทถาวร คุณจึงไม่ต้องเสียโทเคนอธิบายซ้ำทุกบทสนทนา

สร้าง Project สำหรับแต่ละสัญญางานหรือสายงานระยะยาว เช่น งานติดตามและวิเคราะห์ การศึกษาความเสี่ยงห่วงโซ่อุปทาน การพัฒนาผลิตภัณฑ์ SAM หรือโครงการทำแผนผัง โดยเก็บบริบทคงที่ไว้ในความรู้/คำสั่งของ Project เพียงครั้งเดียว:

  • วัตถุประสงค์ ขอบเขต และคำศัพท์ที่ลูกค้าอนุมัติ (ปกปิดตัวตนตามระดับที่กฎ Tier อนุญาต)
  • บันทึกระเบียบวิธีของเรา เช่น วิธีตรวจสอบแหล่งข้อมูล เกณฑ์ให้คะแนน มาตรฐานการแสดงผลข้อมูล
  • แม่แบบผลงาน: โครงสร้างรายงาน รูปแบบการอ้างอิง ข้อจำกัดมาตรฐาน
  • คำอธิบาย schema ของชุดข้อมูลที่เกี่ยวข้อง (เฉพาะโครงสร้าง ไม่มีข้อมูลจริง)
ดี — บริบทอยู่ใน Project

คำสั่งของ Project มีแม่แบบรายงานและระเบียบวิธีอยู่แล้ว แชตใหม่แต่ละครั้ง: "ร่างส่วนแนวโน้ม Q2 จากข้อความที่แนบ" ใช้พรอมต์เพียง ~200 โทเคน

สิ้นเปลือง — วางบริบทซ้ำทุกครั้ง

ทุกแชตเปิดด้วยข้อมูลพื้นหลัง 3,000 คำ แม่แบบฉบับเต็ม และรายงานไตรมาสก่อน ก่อนถึงคำขอจริง

วินัยการใช้โฟลเดอร์ใน Cowork

Cowork ทำงานกับโฟลเดอร์ที่คุณเชื่อมต่ออย่างชัดเจน จงปฏิบัติต่อการเชื่อมต่อนั้นเหมือนการส่งมอบตู้เอกสารทั้งตู้:

  • หนึ่งโฟลเดอร์งานต่อหนึ่งสัญญางาน สร้างโฟลเดอร์ทำงานเฉพาะที่มีแต่ไฟล์ที่งานต้องใช้ เชื่อมต่อเฉพาะโฟลเดอร์นั้น — ไม่ใช่โฟลเดอร์แม่ เดสก์ท็อป หรือโฟลเดอร์ซิงก์ Drive ทั้งหมด
  • ห้ามโฟลเดอร์งานส่งมอบลูกค้า งานส่งมอบลับฉบับสมบูรณ์อยู่ในที่จัดเก็บที่ป้องกันไว้ ซึ่งไม่เชื่อมต่อกับ Cowork เด็ดขาด ทำงานร่างในโฟลเดอร์งาน แล้วย้ายฉบับสมบูรณ์ออกเมื่อเสร็จ
  • คัดกรองก่อนเชื่อมต่อ ก่อนเชื่อมต่อโฟลเดอร์ ตรวจดูว่าไฟล์ใดระบุชื่อแหล่งข่าว เจ้าหน้าที่ภาคสนาม หรือบุคคลเป้าหมายที่งานไม่ต้องใช้ ให้ย้ายออกก่อน
  • ตรวจสอบสิ่งที่ Cowork สร้าง ก่อนส่งไฟล์ต่อ คุณคือด่านคุณภาพ เหมือนตรวจงานนักวิเคราะห์รุ่นน้อง

กฎการเชื่อมต่อ

การเชื่อมต่อสถานะเหตุผล
อีเมล (Gmail / Outlook)ห้ามอีเมลมีจดหมายลูกค้า สัญญา และผู้ติดต่อแหล่งข่าว ห้ามเครื่องมือ AI ใดเชื่อมต่อกับอีเมลของ ENVISAGE เด็ดขาด
Google Driveจำกัดเฉพาะโฟลเดอร์ทำงานที่อนุมัติล่วงหน้า — ห้ามเข้าถึงระดับราก โฟลเดอร์งานส่งมอบลูกค้า ฝ่ายบุคคล และการเงิน ถูกยกเว้นถาวร ต้องได้รับอนุมัติจากหัวหน้าโครงการ
การแชร์เอกสาร / สเปรดชีตห้ามห้ามแชร์ลิงก์ Docs/Sheets ภายในกับเครื่องมือ AI ใด ๆ และห้ามวางเอกสารลับฉบับเต็ม ใช้ข้อความที่คัดกรองแล้วหรือโฟลเดอร์งานใน Cowork
โฟลเดอร์งานในเครื่อง (Cowork)อนุมัติโฟลเดอร์เฉพาะกิจที่มีแต่ไฟล์ที่เกี่ยวข้องกับงาน ตามวินัยข้างต้น

รูปแบบพรอมต์สำหรับงานวิจัย

ระบุบทบาท แหล่งข้อมูล และรูปแบบผลลัพธ์ — ครั้งเดียว

คุณกำลังช่วยนักวิเคราะห์ของ ENVISAGE ศึกษาห่วงโซ่อุปทาน
แหล่งข้อมูล: ข้อความด้านล่าง (ข่าวสาธารณะ แปลแล้ว)
งาน: ดึงชื่อบริษัท สถานที่ และพฤติกรรมที่ถูกกล่าวหา
เป็นตาราง 3 คอลัมน์ ระบุรายการที่ความเชื่อมั่นต่ำ

ให้ Claude ถามก่อนสร้างผลลัพธ์ยาว

สำหรับงานชิ้นใหญ่ เพิ่มว่า: "ก่อนร่าง ให้แจกแจงสมมติฐานและโครงร่างภายใน 150 คำ แล้วรอการยืนยัน" การแก้โครงร่าง 100 โทเคนถูกกว่าการสร้างร่าง 3,000 โทเคนใหม่มาก

แก้ทีละส่วน ไม่ใช่ทั้งเอกสาร

ขอแก้เฉพาะส่วนที่ต้องปรับ ("แก้เฉพาะย่อหน้า 2 — ส่วนอื่นคงเดิม") ไม่ใช่สร้างใหม่ทั้งหมด

အပိုင်း ၁

Claude နှင့် Claude Cowork အကောင်းဆုံးအလေ့အကျင့်များ

Claude သည် ကျွန်ုပ်တို့၏ အဓိက AI ကိရိယာဖြစ်သည်။ စဉ်းစားရန် Chat၊ လုပ်ဆောင်ရန် Cowork ကိုသုံးပါ — သုတေသနလိုင်းတစ်ခုစီကို တစ်ကြိမ်တည်းတည်ဆောက်ထားလျှင် နောက်ခံအကြောင်းအရာကို ထပ်ခါထပ်ခါရှင်းပြရန် မလိုတော့ပါ။

Chat နှင့် Cowork — မှန်ကန်သောမုဒ်ကို ရွေးပါ

Claude Chat

စဉ်းစားခြင်း၊ မူကြမ်းရေးခြင်း၊ ခွဲခြမ်းစိတ်ဖြာခြင်းအတွက်

  • သုတေသနမေးခွန်း သို့မဟုတ် ခွဲခြမ်းစိတ်ဖြာမှုဘောင်ကို ဆင်ခြင်ခြင်း
  • အစီရင်ခံစာအပိုင်းများ၊ အကျဉ်းချုပ်များ၊ အကျဉ်းတင်ပြချက်များ ရေးခြင်းနှင့် ပြင်ခြင်း
  • မြန်မာ၊ ထိုင်း၊ ရှမ်းနှင့် အခြားဘာသာစကားများမှ ဘာသာပြန်ချက်များ စစ်ဆေးခြင်း
  • ဖော်ပြထားသော သို့မဟုတ် ကောက်နုတ်ထည့်ထားသော ဒေတာများတွင် ပုံစံများကို အဓိပ္ပာယ်ဖွင့်ဆိုခြင်း
  • အင်တာဗျူးမှတ်စုများ၊ ကွင်းဆင်းမှတ်တမ်းများ ဖွဲ့စည်းခြင်း (အမည်ဖျောက်ထားသော)
Claude Cowork

ဖိုင်များပေါ်တွင် တိုက်ရိုက်လုပ်ဆောင်ရန်

  • ပေးအပ်စာတမ်းများ တည်ဆောက်ခြင်း — Word အစီရင်ခံစာ၊ Excel၊ ဆလိုက်များ
  • ဒေတာဖိုင်များ သန့်စင်ခြင်း၊ ပြန်လည်ဖွဲ့စည်းခြင်း၊ ပြောင်းလဲခြင်း
  • အဆင့်များစွာပါအလုပ်များ — "ဤ CSV ၁၂ ဖိုင်ကို စီမံပြီး အကျဉ်းချုပ်ဇယားထုတ်ပါ"
  • ပရောဂျက်ဖိုင်တွဲအတွင်း ဖိုင်များစွာတွင် ထပ်တလဲလဲ စာရွက်စာတမ်းအလုပ်များ
  • နမူနာဖိုင်များဖြင့် script (Python, SQL) ရေးခြင်းနှင့် စမ်းသပ်ခြင်း

လက်တွေ့စည်းမျဉ်း — အဖြေသည် ဖိုင်ထဲမှာရှိသင့်လျှင် Cowork မှာစတင်ပါ။ အဖြေသည် သင့်ဦးနှောက်ထဲမှာရှိသင့်လျှင် Chat မှာစတင်ပါ။ ဖိုင်အကြောင်းအရာများကို Chat ထဲကူးထည့်ခြင်းသည် တိုကင်နှစ်ဆဖြုန်းသည် — ကူးထည့်စဉ်တစ်ကြိမ်၊ Claude က ပြန်ရေးပြစဉ်တစ်ကြိမ်။

သုတေသနလိုင်းတစ်ခုလျှင် Project တစ်ခု တည်ဆောက်ပါ

Claude Projects သည် တည်မြဲသောအကြောင်းအရာကို သိမ်းဆည်းထားသဖြင့် စကားပြောတိုင်းတွင် ထပ်ရှင်းပြရန် မလိုတော့ပါ။

ရေရှည်စာချုပ် သို့မဟုတ် လုပ်ငန်းလိုင်းတစ်ခုစီအတွက် Project တစ်ခုဖန်တီးပါ — ဥပမာ စောင့်ကြည့်လေ့လာရေးစာချုပ်၊ ထောက်ပံ့ရေးကွင်းဆက်အန္တရာယ်လေ့လာမှု၊ SAM ထုတ်ကုန်ဖွံ့ဖြိုးရေး သို့မဟုတ် မြေပုံရေးဆွဲရေးပရောဂျက်။ Project ၏ knowledge/instructions တွင် တည်ငြိမ်သောအကြောင်းအရာကို တစ်ကြိမ်တည်း သိမ်းပါ —

  • စာချုပ်၏ရည်ရွယ်ချက်၊ နယ်ပယ်နှင့် ဖောက်သည်အတည်ပြု ဝေါဟာရများ (Tier စည်းမျဉ်းခွင့်ပြုသည့်အဆင့်အထိ အမည်ဖျောက်၍)
  • ကျွန်ုပ်တို့၏နည်းစနစ်မှတ်စုများ — သတင်းရင်းမြစ်စစ်ဆေးနည်း၊ အမှတ်ပေးစံ၊ ပုံဖော်မှုစံနှုန်းများ
  • ထုတ်ကုန်ပုံစံများ — အစီရင်ခံစာဖွဲ့စည်းပုံ၊ ကိုးကားပုံစံ၊ စံသတိပေးချက်များ
  • သက်ဆိုင်ရာဒေတာအစုများ၏ schema ဖော်ပြချက် (ဖွဲ့စည်းပုံသာ၊ ဒေတာအတန်းမပါ)
ကောင်း — အကြောင်းအရာသည် Project ထဲမှာရှိသည်

Project instructions တွင် အစီရင်ခံစာပုံစံနှင့် နည်းစနစ်ပါပြီးသား။ စကားပြောအသစ်တိုင်း — "ပူးတွဲကောက်နုတ်ချက်ကိုသုံး၍ Q2 လမ်းကြောင်းအပိုင်း မူကြမ်းရေးပါ" — prompt တိုကင် ~၂၀၀ သာ။

ဖြုန်းတီး — အကြောင်းအရာကို အကြိမ်တိုင်း ပြန်ကူးထည့်

စကားပြောတိုင်း နောက်ခံစာလုံး ၃,၀၀၀၊ ပုံစံအပြည့်အစုံနှင့် ယခင်သုံးလပတ်အစီရင်ခံစာကို အမှန်တကယ်တောင်းဆိုချက်မတိုင်မီ ကူးထည့်သည်။

Cowork ဖိုင်တွဲ စည်းကမ်း

Cowork သည် သင်ချိတ်ဆက်ပေးသော ဖိုင်တွဲများပေါ်တွင် အလုပ်လုပ်သည်။ ထိုချိတ်ဆက်မှုကို ဖိုင်ဗီရိုတစ်လုံးလုံး လွှဲအပ်သလို သဘောထားပါ —

  • စာချုပ်တစ်ခုလျှင် အလုပ်ဖိုင်တွဲတစ်ခု။ အလုပ်လိုအပ်သောဖိုင်များသာပါသည့် သီးသန့်ဖိုင်တွဲဖန်တီးပြီး ၎င်းကိုသာချိတ်ပါ — မိခင်ဖိုင်တွဲ၊ Desktop သို့မဟုတ် Drive sync ဖိုင်တွဲတစ်ခုလုံး ဘယ်တော့မှမချိတ်ရ။
  • ဖောက်သည်ပေးအပ်စာတမ်းဖိုင်တွဲများ မချိတ်ရ။ နောက်ဆုံးလျှို့ဝှက်ပေးအပ်စာတမ်းများသည် Cowork နှင့်ဘယ်တော့မှမချိတ်သော ကာကွယ်ထားသည့်နေရာတွင် ရှိရမည်။ မူကြမ်းများကို အလုပ်ဖိုင်တွဲတွင်လုပ်ပြီး ပြီးလျှင် နောက်ဆုံးမူများကို ရွှေ့ထုတ်ပါ။
  • မချိတ်ခင် စစ်ထုတ်ပါ။ ဖိုင်တွဲမချိတ်ခင် စစ်ဆေးပါ — အလုပ်မလိုအပ်သော သတင်းရင်းမြစ်၊ ကွင်းဆင်းဝန်ထမ်း သို့မဟုတ် စိတ်ဝင်စားရသူပုဂ္ဂိုလ်နာမည်ပါသည့် ဖိုင်များကို အရင်ရွှေ့ထုတ်ပါ။
  • Cowork ထုတ်လုပ်သည့်အရာများကို စစ်ဆေးပါ — ဖိုင်များကို ရှေ့မဆက်ခင်။ အငယ်တန်းသုံးသပ်သူ၏အလုပ်ကို စစ်သလိုပင် သင်သည် အရည်အသွေးတံခါးဖြစ်သည်။

ချိတ်ဆက်မှု စည်းမျဉ်းများ

ချိတ်ဆက်မှုအခြေအနေအကြောင်းရင်း
အီးမေးလ် (Gmail / Outlook)တားမြစ်အီးမေးလ်တွင် ဖောက်သည်စာပေးစာယူ၊ စာချုပ်များနှင့် သတင်းရင်းမြစ်အဆက်အသွယ်များ ပါသည်။ မည်သည့် AI ကိရိယာမျှ ENVISAGE အီးမေးလ်နှင့် ဘယ်တော့မှ မချိတ်ရ။
Google Driveကန့်သတ်ကြိုတင်ခွင့်ပြုထားသော သီးသန့်အလုပ်ဖိုင်တွဲများသာ — root အဆင့် ဘယ်တော့မှမခွင့်ပြုရ။ ဖောက်သည်ပေးအပ်စာတမ်း၊ HR နှင့် ငွေကြေးဖိုင်တွဲများ အမြဲချန်လှပ်ထားသည်။ ပရောဂျက်ခေါင်းဆောင်၏ ခွင့်ပြုချက်လိုသည်။
စာတမ်း / spreadsheet မျှဝေခြင်းတားမြစ်အတွင်းပိုင်း Docs/Sheets လင့်ခ်များကို မည်သည့် AI ကိရိယာနှင့်မျှ မမျှဝေရ၊ လျှို့ဝှက်စာတမ်းအပြည့်အစုံ မကူးထည့်ရ။ သန့်စင်ထားသောကောက်နုတ်ချက် သို့မဟုတ် Cowork အလုပ်ဖိုင်တွဲများကို သုံးပါ။
စက်တွင်း အလုပ်ဖိုင်တွဲများ (Cowork)ခွင့်ပြုအထက်ပါစည်းကမ်းအတိုင်း အလုပ်နှင့်သက်ဆိုင်သောဖိုင်များသာပါသည့် ရည်ရွယ်ချက်သီးသန့်ဖိုင်တွဲများ။

သုတေသနအလုပ်အတွက် Prompt ပုံစံများ

အခန်းကဏ္ဍ၊ ရင်းမြစ်၊ ထုတ်ကုန်ပုံစံကို တစ်ကြိမ်တည်း ပေးပါ

သင်သည် ထောက်ပံ့ရေးကွင်းဆက်လေ့လာမှုတွင် ENVISAGE သုံးသပ်သူကို ကူညီနေသည်။
ရင်းမြစ် — အောက်ပါကောက်နုတ်ချက် (အများပြည်သူသတင်း၊ ဘာသာပြန်ပြီး)။
အလုပ် — ကုမ္ပဏီနာမည်၊ တည်နေရာနှင့် စွပ်စွဲခံရသောလုပ်ရပ်များကို
ကော်လံ ၃ ခုပါဇယားအဖြစ် ထုတ်ပါ။ ယုံကြည်မှုနည်းသောအချက်များကို မှတ်သားပါ။

ရှည်လျားသောထုတ်ကုန် မထုတ်ခင် Claude ကို မေးခိုင်းပါ

ကြီးမားသောပေးအပ်စာတမ်းများအတွက် ထည့်ပါ — "မူကြမ်းမရေးခင် သင့်ယူဆချက်များနှင့် အကြမ်းဖျင်းကို စာလုံး ၁၅၀ အောက်ဖြင့် ဖော်ပြပြီး ကျွန်ုပ်၏အတည်ပြုချက်ကို စောင့်ပါ။" တိုကင် ၁၀၀ တန် အကြမ်းဖျင်းပြင်ဆင်မှုသည် တိုကင် ၃,၀၀၀ တန် မူကြမ်းပြန်ထုတ်ခြင်းထက် များစွာသက်သာသည်။

စာတမ်းတစ်ခုလုံးမဟုတ်ဘဲ အပိုင်းအလိုက် ပြင်ပါ

ပြင်ရန်လိုသောအပိုင်းကိုသာ တောင်းပါ ("စာပိုဒ် ၂ ကိုသာပြင်ပါ — ကျန်အားလုံးထားပါ")၊ အစအဆုံးပြန်ထုတ်ခိုင်းခြင်း မဟုတ်ပါ။
