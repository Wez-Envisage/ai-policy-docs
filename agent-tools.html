AI Agent Tools & Platforms: Capabilities and Considerations

A plain-language overview of the AI agent tools available to ENVISAGE teams, with a structured pros and cons matrix to guide platform decisions for the Sapphire and SAM programmes.

English — AI Agent Tools Briefing & Comparison Matrix

Background

As ENVISAGE teams explore how to connect AI tools to live data — broker notes, research databases, client files — it is important to understand what each platform actually does, what it can access, and what risks it introduces. This briefing summarises the six key tools under consideration and provides a structured comparison to guide decisions.

The central question is not which tool is most capable, but which tool gives ENVISAGE the right combination of control, security, and flexibility — particularly given the sensitive nature of our client data and the long-term ambition of SamAI as our own decision-making infrastructure.

Security Context: Any tool that connects to your computer, file system, or cloud storage is, to some degree, giving the AI provider access to that data. The tools below vary significantly in how much access they require, how transparent they are about it, and how much control you retain. This must be weighed carefully before connecting any tool to confidential client data.

The Six Platforms at a Glance

🖥

Claude Cowork

Desktop AI Workspace · Anthropic

A desktop application that connects Claude to files, folders, and external services via MCP (Model Context Protocol). It presents as a clean, user-friendly interface, but underneath it functions as a UI connecting directly to Anthropic's cloud infrastructure. Once connected to a folder or Google Drive, the AI can read and act on those files.

⌨️

Claude Code

CLI Coding Agent · Anthropic

A command-line tool that lets developers run Claude directly from the terminal to write, edit, and debug code. It can be granted access to specific files and directories. More transparent than Cowork for technical users — the developer explicitly controls what Claude sees. Confirmed compatible with ENVISAGE's SAM environment.

🔓

OpenCode

Open-Source CLI Coding Agent

An open-source alternative to Claude Code and Codex. It can work with multiple LLMs and does not lock you into a single provider. Because the source code is publicly auditable, security-conscious teams can inspect exactly what data is transmitted. Confirmed compatible with SAM.

🤖

OpenAI Codex

CLI Coding Agent · OpenAI

OpenAI's equivalent of Claude Code — a terminal-based coding agent that can read files, write code, and execute tasks. Operates within a sandboxed environment but still transmits data to OpenAI's infrastructure. Confirmed compatible with SAM. A strong alternative when Claude is unavailable or for cross-verification tasks.

🗄️

Pi Agent (NL-to-SQL)

Natural Language to SQL Agent

A category of coding agent that translates plain-language questions into SQL database queries. The user describes what they want to find; the agent generates the query. Particularly useful for non-developer staff who need to extract data from SAM's databases (PostgreSQL, Snowflake, Elasticsearch) without writing code. Confirmed compatible with SAM.

🏗️

SamAI

ENVISAGE Proprietary Platform

ENVISAGE's own AI decision-making infrastructure. Unlike the other tools, SamAI is designed to be LLM-agnostic — any model can plug in and operate within its structure. It places an explicit control layer between the LLM and ENVISAGE's databases, meaning the AI only sees what ENVISAGE deliberately chooses to expose. Skills, tools, and decision logic are owned and controlled entirely by ENVISAGE.

Pros & Cons by Platform

✅ Claude Cowork — Strengths

  • Excellent UI — accessible to non-technical staff
  • Broad MCP connector library (Google Drive, Slack, etc.)
  • Persistent context across sessions via Projects
  • Already used and tested by ENVISAGE team

⚠️ Claude Cowork — Risks & Limits

  • Fundamentally gives Anthropic access to connected files and folders
  • Prompt injection vulnerabilities are not fully solved
  • Limited transparency for non-developers about what data is shared
  • Locks you into Claude / Anthropic infrastructure
  • Not suitable for connecting to confidential client deliverables

✅ Claude Code — Strengths

  • Developer-controlled — explicit file and directory permissions
  • Highly capable for code generation, debugging, and data tasks
  • Can be limited to specific files or read-only access
  • Confirmed compatible with SAM environment

⚠️ Claude Code — Risks & Limits

  • Requires technical setup — not suitable for non-developer staff
  • Data still transmitted to Anthropic's infrastructure
  • LLM-locked to Claude/Anthropic

✅ OpenCode — Strengths

  • Open-source and auditable — full transparency on data handling
  • Works with multiple LLMs — not locked to any single provider
  • Can be self-hosted to keep data entirely in-house
  • Confirmed compatible with SAM

⚠️ OpenCode — Risks & Limits

  • Less polished UI — requires technical confidence to set up
  • Smaller community and support ecosystem
  • Requires more initial configuration than commercial tools

✅ OpenAI Codex — Strengths

  • Strong coding capabilities — reliable for complex tasks
  • Good alternative when Claude is at capacity or for cross-checking
  • Sandboxed execution environment reduces some risk
  • Confirmed compatible with SAM

⚠️ OpenAI Codex — Risks & Limits

  • Data transmitted to OpenAI's infrastructure
  • LLM-locked to OpenAI
  • Paid service — cost scales with usage
  • Privacy policy differences from Anthropic — review before use with client data

✅ Pi Agent (NL-to-SQL) — Strengths

  • Makes database querying accessible to non-technical staff
  • Schema-in / query-out pattern keeps AI away from raw data
  • Dramatically reduces time to extract structured insights from SAM databases
  • Confirmed compatible with SAM (PostgreSQL, Snowflake, Elasticsearch)

⚠️ Pi Agent — Risks & Limits

  • Database schema must be carefully controlled — do not expose full schema
  • Generated SQL queries must be reviewed before execution on production data
  • Quality varies — complex multi-table queries may require human correction

✅ SamAI — Strengths

  • LLM-agnostic — any model can be swapped in without rebuilding
  • ENVISAGE owns the skills, tools, and decision logic entirely
  • Explicit access control layer between LLM and databases
  • Purpose-built for ENVISAGE's research and investment workflows
  • No external provider has access to client data beyond what ENVISAGE defines
  • Continuously improvable — skills and features evolve with team needs

⚠️ SamAI — Risks & Limits

  • Still being built — some features require development time
  • Requires internal technical expertise to extend and maintain
  • Higher upfront investment than using off-the-shelf tools
  • Team must be disciplined about not bypassing the control layer

Recommended Approach

For day-to-day staff work (Tier 1 & 2 data): Claude Cowork and Claude Chat remain appropriate for research, summarisation, and analysis of non-confidential material. Follow the data tier rules in the ENVISAGE AI Policy.

For database access and structured data extraction: Use Pi Agent (NL-to-SQL) via the schema-in / query-out pattern. The AI sees the structure, not the raw data. All generated queries should be reviewed before execution.

For any task involving client deliverables or Tier 3 data: Do not use any third-party AI tool. Route through SamAI where the access control layer protects sensitive data by design. If SamAI does not yet support a task, escalate to Simon or the technical team — do not improvise with external tools.

Long-term direction: As SamAI matures, it should become the primary interface for all data-connected AI work at ENVISAGE. The decision to remain LLM-agnostic is intentional — it prevents vendor lock-in and ensures ENVISAGE's skills and decision logic remain our own intellectual property regardless of which model we use.

ภาษาไทย — บทสรุปเครื่องมือ AI Agent และเมทริกซ์เปรียบเทียบ

ภูมิหลัง

เมื่อทีม ENVISAGE สำรวจวิธีเชื่อมต่อเครื่องมือ AI กับข้อมูลสด — บันทึกนายหน้า ฐานข้อมูลวิจัย ไฟล์ลูกค้า — สิ่งสำคัญคือต้องเข้าใจว่าแต่ละแพลตฟอร์มทำอะไรได้จริง เข้าถึงอะไรได้ และมีความเสี่ยงอะไร

คำถามหลักไม่ใช่เครื่องมือใดมีความสามารถมากที่สุด แต่คือเครื่องมือใดให้ ENVISAGE ได้รับการควบคุม ความปลอดภัย และความยืดหยุ่นที่เหมาะสม โดยเฉพาะเมื่อพิจารณาถึงความอ่อนไหวของข้อมูลลูกค้าของเราและความทะเยอทะยานระยะยาวของ SamAI

บริบทความปลอดภัย: เครื่องมือใดก็ตามที่เชื่อมต่อกับคอมพิวเตอร์ ระบบไฟล์ หรือที่เก็บข้อมูลบนคลาวด์ของคุณ จะให้ผู้ให้บริการ AI เข้าถึงข้อมูลนั้นในระดับหนึ่ง ต้องชั่งน้ำหนักอย่างรอบคอบก่อนเชื่อมต่อเครื่องมือใดๆ กับข้อมูลลูกค้าที่เป็นความลับ

หกแพลตฟอร์มโดยสรุป

🖥

Claude Cowork

พื้นที่ทำงาน AI ตั้งโต๊ะ · Anthropic

แอปพลิเคชันตั้งโต๊ะที่เชื่อมต่อ Claude กับไฟล์ โฟลเดอร์ และบริการภายนอกผ่าน MCP ดูเป็นอินเทอร์เฟซที่เรียบง่าย แต่ในความเป็นจริงคือ UI ที่เชื่อมต่อโดยตรงกับโครงสร้างพื้นฐานคลาวด์ของ Anthropic

⌨️

Claude Code

ตัวแทนเขียนโค้ด CLI · Anthropic

เครื่องมือบรรทัดคำสั่งที่ให้นักพัฒนาใช้ Claude จากเทอร์มินัลเพื่อเขียน แก้ไข และดีบักโค้ด นักพัฒนาควบคุมโดยตรงว่า Claude เห็นอะไร ยืนยันว่าใช้งานร่วมกับสภาพแวดล้อม SAM ได้

🔓

OpenCode

ตัวแทนเขียนโค้ด CLI โอเพนซอร์ส

ทางเลือกโอเพนซอร์สสำหรับ Claude Code และ Codex ทำงานร่วมกับหลาย LLM และไม่ผูกคุณกับผู้ให้บริการรายเดียว โค้ดต้นฉบับสามารถตรวจสอบได้ ยืนยันว่าใช้งานร่วมกับ SAM ได้

🤖

OpenAI Codex

ตัวแทนเขียนโค้ด CLI · OpenAI

เทียบเท่ากับ Claude Code ของ OpenAI — ตัวแทนเขียนโค้ดผ่านเทอร์มินัล ยืนยันว่าใช้งานร่วมกับ SAM ได้ เป็นทางเลือกที่ดีเมื่อ Claude ไม่พร้อมใช้หรือสำหรับการตรวจสอบข้าม

🗄️

Pi Agent (NL-to-SQL)

ตัวแทนแปลงภาษาธรรมชาติเป็น SQL

ประเภทของตัวแทนที่แปลคำถามภาษาธรรมดาเป็นคำสั่ง SQL สำหรับฐานข้อมูล ผู้ใช้อธิบายสิ่งที่ต้องการค้นหา ตัวแทนสร้างคำสั่ง ยืนยันว่าใช้งานร่วมกับ SAM ได้ (PostgreSQL, Snowflake, Elasticsearch)

🏗️

SamAI

แพลตฟอร์มของ ENVISAGE

โครงสร้างพื้นฐานการตัดสินใจด้วย AI ของ ENVISAGE เอง ออกแบบให้ไม่ผูกกับ LLM ใด วาง layer ควบคุมที่ชัดเจนระหว่าง LLM และฐานข้อมูลของ ENVISAGE ทักษะ เครื่องมือ และตรรกะการตัดสินใจเป็นของ ENVISAGE อย่างสมบูรณ์

แนวทางที่แนะนำ

สำหรับงานเจ้าหน้าที่ประจำวัน (ข้อมูลระดับ 1 และ 2): Claude Cowork และ Claude Chat ยังเหมาะสมสำหรับการวิจัย สรุป และวิเคราะห์วัสดุที่ไม่เป็นความลับ ปฏิบัติตามกฎระดับข้อมูลในนโยบาย AI ของ ENVISAGE

สำหรับการเข้าถึงฐานข้อมูล: ใช้ Pi Agent (NL-to-SQL) ผ่านรูปแบบ Schema-in / query-out AI เห็นเฉพาะโครงสร้าง ไม่ใช่ข้อมูลดิบ ควรตรวจสอบคำสั่ง SQL ที่สร้างขึ้นก่อนประมวลผล

สำหรับงานที่เกี่ยวข้องกับผลงานลูกค้าหรือข้อมูลระดับ 3: ห้ามใช้เครื่องมือ AI ของบุคคลที่สาม ใช้ผ่าน SamAI ที่มี layer ควบคุมการเข้าถึงปกป้องข้อมูลที่อ่อนไหวโดยออกแบบ

ทิศทางระยะยาว: เมื่อ SamAI พัฒนาขึ้น ควรกลายเป็นอินเทอร์เฟซหลักสำหรับงาน AI ทั้งหมดที่เชื่อมต่อข้อมูลที่ ENVISAGE การตัดสินใจไม่ผูกกับ LLM เป็นเจตนาเพื่อป้องกันการผูกกับผู้ขายและรับรองว่าทักษะและตรรกะการตัดสินใจของ ENVISAGE ยังคงเป็นทรัพย์สินทางปัญญาของเราเอง

နောက်ခံ

ENVISAGE အဖွဲ့များ AI ကိရိယာများကို တိုက်ရိုက်ဒေတာ — ဒိုင်ကွမ်းမှတ်တမ်းများ၊ သုတေသနဒေတာဘေ့စ်များ၊ ဖောက်သည်ဖိုင်များ — နှင့် မည်သို့ချိတ်ဆက်မည်ကို စူးစမ်းသောအချိန်တွင် ပလက်ဖောင်းတစ်ခုချင်းစီ အမှန်တကယ်ဘာလုပ်သည်၊ ဘာကိုအသုံးပြုနိုင်သည်၊ ဘာအန္တရာယ်မြောက်ဆောင်လာသည်ကို နားလည်ရန် အရေးကြီးသည်။

အဓိကမေးခွန်းမှာ မည်သည့်ကိရိယာ အစွမ်းထက်ဆုံးသည်မဟုတ်ဘဲ မည်သည့်ကိရိယာ ENVISAGE ကို ထိန်းချုပ်မှု၊ လုံခြုံရေးနှင့် ပြောင်းလွယ်ပြင်လွယ်မှု မှန်ကန်သောပေါင်းစပ်ချက်ပေးသည်ဆိုသည်ဖြစ်သည်။

လုံခြုံရေးအကြောင်းအရာ: သင်၏ကွန်ပျူတာ၊ ဖိုင်စနစ် သို့မဟုတ် cloud သိုလှောင်မှုနှင့် ချိတ်ဆက်သောကိရိယာတစ်ခုမဆို AI ပံ့ပိုးသူကို ဒေတာကိုင်တွယ်ရန် ပေးသောအချက်ဖြစ်သည်။ လျှို့ဝှက်ဖောက်သည်ဒေတာနှင့် မချိတ်ဆက်မီ ဤအချက်ကို သေချာစဉ်းစားရမည်။

SamAI

ENVISAGE ပိုင်ပလက်ဖောင်း

ENVISAGE ၏ကိုယ်ပိုင် AI ဆုံးဖြတ်ချက်ချသောအခြေခံဆောက်လုပ်ရေး။ LLM-agnostic ဒီဇိုင်းဖြင့် ဆောက်လုပ်ထားပြီး LLM နှင့် ENVISAGE ၏ ဒေတာဘေ့စ်ကြား ထိန်းချုပ်ရေး layer ထားသည်။ Skills၊ Tools နှင့် ဆုံးဖြတ်ချက်ချသောယုတ္တိ ENVISAGE ပိုင်ဆိုင်သည်။

ပြည့်စုံသောနှိုင်းယှဉ်မက်ထရစ်

အတိုင်းအတာ Claude Cowork Claude Code OpenCode Codex (OpenAI) Pi Agent SamAI

အကြံပြုထားသောချဉ်းကပ်နည်း

နေ့စဉ်ဝန်ထမ်းလုပ်ငန်းများအတွက် (Tier 1 & 2 ဒေတာ): Claude Cowork နှင့် Claude Chat သည် လျှို့ဝှက်မဟုတ်သောသုတေသန၊ အကျဉ်းချုပ်နှင့် ခွဲခြမ်းစိတ်ဖြာမှုအတွက် သင့်တော်သည်။ ENVISAGE AI မူဝါဒရှိ ဒေတာ tier စည်းကမ်းများ လိုက်နာပါ။

ဒေတာဘေ့စ်ကိုင်တွယ်မှုအတွက်: Schema-in / query-out ပုံစံမှတဆင့် Pi Agent (NL-to-SQL) သုံးပါ။ AI သည် ဖွဲ့စည်းပုံသာမြင်သည်၊ ကြမ်းဒေတာမဟုတ်။ SQL command များ execute မလုပ်မီ စစ်ဆေးပါ။

ဖောက်သည်ရလဒ်များ သို့မဟုတ် Tier 3 ဒေတာပါဝင်သောအလုပ်အတွက်: မည်သည့် ပြင်ပ AI ကိရိယာမျှ မသုံးပါနှင့်။ ဒေတာကို ဒီဇိုင်းအရ ကာကွယ်သော ထိန်းချုပ်ရေး layer ပါသည့် SamAI မှတဆင့် လမ်းကြောင်းလွှဲပါ။

ရေရှည်လမ်းညွှန်ချက်: SamAI ကြီးထွားလာသောအချိန်တွင် ENVISAGE ရှိ ဒေတာချိတ်ဆက်သော AI လုပ်ငန်းအားလုံးအတွက် အဓိကမျက်နှာဆိုင်ရာဖြစ်လာသင့်သည်။ LLM-agnostic ဆုံးဖြတ်ချက်သည် ဝန်ဆောင်မှုပြားချည်မှု တားဆီးပြီး ENVISAGE ၏ skills နှင့် ဆုံးဖြတ်ချက်ချသောယုတ္တိ ကျွန်ုပ်တို့၏ ဉာဏပိုင်ဆိုင်မှုအဖြစ် ကျန်ရှိနေကြောင်း သေချာစေသည်။
