Section 5

The Wider AI Toolkit

Free public LLMs and specialized coding agents let us reserve paid Claude capacity for the work that needs it. The trade-off is control: the cheaper the tool, the less sensitive the content it may touch.

Free Public LLMs: DeepSeek & Kimi

DeepSeek (DeepSeek AI) and Kimi (Moonshot AI) are capable free models — strong at coding, long-document handling, and general drafting, often rivaling paid models on generic tasks. They are excellent token-savers for Tier 1 work.

Hard limit: Tier 1 only. These services are hosted outside our control, with data-handling terms we cannot enforce — and given where ENVISAGE operates and what we research, our subject matter is uniquely sensitive in that context. Nothing internal, nothing client-related, no names from our research — ever. Assume everything sent is retained and readable.

Good uses (Tier 1)

  • Summarizing published news articles and public reports during open-source harvesting triage
  • First-pass generic drafting: boilerplate descriptions, meeting agendas, generic outreach templates
  • Explaining a public concept, statistical method, or library before you spend Claude tokens applying it to our context
  • Generic code snippets and debugging help (no schema names, no internal code)
  • Brainstorming structures and checklists with no ENVISAGE specifics
Good routing

An analyst triaging 30 public articles asks Kimi for two-sentence summaries of each, picks the 6 relevant ones, then brings only those into Claude for contextual analysis against the engagement. Free tool does the bulk reading; Claude does the judgment.

Never

Pasting a draft client assessment into DeepSeek "just to tighten the wording." The wording is attached to Tier 3 content — that draft now lives on infrastructure we don't control.

Coding Agents: Plain Language to SQL and Beyond

Natural-language-to-SQL agents (Pi-style NL2SQL tools, SQL Agent CLI, and similar) and coding assistants turn plain-English requests into working queries and scripts. For a team whose stack runs on Python, PostgreSQL, Elasticsearch, and Snowflake, they let analysts who don't write SQL get answers without queuing for a developer.

The safe pattern: schema in, query out

A coding agent never needs your data to write your query — it needs the shape of your data. Describe tables and columns; run the generated query yourself inside our environment.

Our PostgreSQL database has:
  entities(entity_id, name_hash, type, country, first_seen)
  relationships(rel_id, entity_a, entity_b, rel_type, confidence)
  mentions(mention_id, entity_id, source_id, published_date)

Write a query returning entities mentioned in 3+ sources
in the last 90 days that have at least one 'ownership'
relationship, ordered by mention count.

Note what the prompt contains: column names and intent — no rows, no real names, no credentials. The agent returns SQL; you review it, cross-check it if it's pipeline-bound, and execute it inside Sapphire.

Rules for coding agents

  • Schema structure only. Table and column names are Tier 2 — fine for approved tools. Data rows are whatever tier the data is, and usually Tier 3.
  • Never paste credentials or connection strings, and never let an external agent connect directly to production databases. Generated queries run through our own access controls.
  • Test on dev data first. AI-generated SQL is confident and frequently subtly wrong (join fan-out, NULL handling, timezone logic).
  • Use Cowork for code that touches our files. When the script must read actual project data, write and run it in Claude Cowork against the task folder — not in an external coding tool.

Token economics

Routing query generation to a free or low-cost coding agent saves Claude capacity twice: the generation itself, and the iteration loop (most queries take 2–3 refinements). Reserve Claude for queries that need engagement context — "flag the pattern we discussed for this client" — which an external agent has no business seeing anyway.

ชุดเครื่องมือ AI ที่กว้างขึ้น

LLM สาธารณะฟรีและเอเจนต์เขียนโค้ดเฉพาะทางช่วยให้เราเก็บความจุ Claude แบบเสียเงินไว้สำหรับงานที่ต้องใช้จริง สิ่งแลกเปลี่ยนคือการควบคุม: เครื่องมือยิ่งถูก เนื้อหาที่แตะได้ยิ่งต้องอ่อนไหวน้อย

LLM สาธารณะฟรี: DeepSeek และ Kimi

DeepSeek (DeepSeek AI) และ Kimi (Moonshot AI) เป็นโมเดลฟรีที่มีความสามารถ — เก่งด้านโค้ด จัดการเอกสารยาว และร่างเนื้อหาทั่วไป มักเทียบเท่าโมเดลเสียเงินในงานทั่วไป เป็นตัวประหยัดโทเคนชั้นดีสำหรับงาน Tier 1

ขีดจำกัดเด็ดขาด: Tier 1 เท่านั้น บริการเหล่านี้โฮสต์อยู่นอกการควบคุมของเรา ด้วยเงื่อนไขจัดการข้อมูลที่เราบังคับใช้ไม่ได้ — และด้วยพื้นที่ที่ ENVISAGE ทำงานและสิ่งที่เราวิจัย เนื้อหาของเราอ่อนไหวเป็นพิเศษในบริบทนั้น ห้ามข้อมูลภายใน ห้ามข้อมูลลูกค้า ห้ามชื่อจากงานวิจัยของเรา — เด็ดขาด ให้ถือว่าทุกสิ่งที่ส่งไปถูกเก็บและอ่านได้

การใช้ที่ดี (Tier 1)

  • สรุปบทความข่าวและรายงานสาธารณะระหว่างคัดกรองข้อมูลโอเพนซอร์ส
  • ร่างแรกของเนื้อหาทั่วไป: คำอธิบายสำเร็จรูป วาระประชุม แม่แบบติดต่อทั่วไป
  • อธิบายแนวคิดสาธารณะ วิธีทางสถิติ หรือไลบรารี ก่อนใช้โทเคน Claude นำไปใช้กับบริบทของเรา
  • โค้ดสั้น ๆ ทั่วไปและช่วยดีบัก (ไม่มีชื่อ schema ไม่มีโค้ดภายใน)
  • ระดมความคิดเรื่องโครงสร้างและเช็กลิสต์โดยไม่มีข้อมูลเฉพาะของ ENVISAGE
การจัดเส้นทางที่ดี

นักวิเคราะห์คัดกรองบทความสาธารณะ 30 ชิ้น ให้ Kimi สรุปชิ้นละสองประโยค เลือก 6 ชิ้นที่เกี่ยวข้อง แล้วนำเฉพาะชิ้นเหล่านั้นเข้า Claude เพื่อวิเคราะห์เชิงบริบทกับสัญญางาน เครื่องมือฟรีอ่านจำนวนมาก Claude ใช้วิจารณญาณ

ห้ามเด็ดขาด

วางร่างรายงานประเมินลูกค้าลง DeepSeek "แค่ขัดเกลาถ้อยคำ" ถ้อยคำนั้นติดอยู่กับเนื้อหา Tier 3 — ร่างนั้นไปอยู่บนโครงสร้างพื้นฐานที่เราไม่ได้ควบคุมแล้ว

เอเจนต์เขียนโค้ด: จากภาษาธรรมดาสู่ SQL และอื่น ๆ

เอเจนต์แปลงภาษาธรรมชาติเป็น SQL (เครื่องมือ NL2SQL แบบ Pi, SQL Agent CLI และที่คล้ายกัน) และผู้ช่วยเขียนโค้ด เปลี่ยนคำขอภาษาธรรมดาเป็นคิวรีและสคริปต์ที่ใช้งานได้ สำหรับทีมที่ใช้ Python, PostgreSQL, Elasticsearch และ Snowflake เครื่องมือเหล่านี้ช่วยให้นักวิเคราะห์ที่เขียน SQL ไม่ได้ ได้คำตอบโดยไม่ต้องรอนักพัฒนา

รูปแบบปลอดภัย: ใส่ schema ได้คิวรี

เอเจนต์เขียนโค้ดไม่ต้องการข้อมูลของคุณเพื่อเขียนคิวรี — มันต้องการรูปร่างของข้อมูล อธิบายตารางและคอลัมน์ แล้วรันคิวรีที่ได้เองภายในระบบของเรา

ฐานข้อมูล PostgreSQL ของเรามี:
  entities(entity_id, name_hash, type, country, first_seen)
  relationships(rel_id, entity_a, entity_b, rel_type, confidence)
  mentions(mention_id, entity_id, source_id, published_date)

เขียนคิวรีที่คืนค่าเอนทิตีที่ถูกกล่าวถึงใน 3+ แหล่ง
ภายใน 90 วันล่าสุด ที่มีความสัมพันธ์ 'ownership'
อย่างน้อยหนึ่งรายการ เรียงตามจำนวนการกล่าวถึง

สังเกตว่าพรอมต์มีอะไร: ชื่อคอลัมน์และเจตนา — ไม่มีข้อมูลจริง ไม่มีชื่อจริง ไม่มีรหัสผ่าน เอเจนต์คืน SQL มา คุณตรวจสอบ ตรวจไขว้ถ้าจะเข้าไปป์ไลน์ แล้วรันภายใน Sapphire

กฎสำหรับเอเจนต์เขียนโค้ด

  • เฉพาะโครงสร้าง schema ชื่อตารางและคอลัมน์เป็น Tier 2 — ใช้กับเครื่องมือที่อนุมัติได้ ข้อมูลจริงเป็น Tier ตามข้อมูลนั้น และมักเป็น Tier 3
  • ห้ามวางรหัสผ่านหรือ connection string และห้ามให้เอเจนต์ภายนอกเชื่อมต่อฐานข้อมูล production โดยตรง คิวรีที่สร้างต้องรันผ่านการควบคุมการเข้าถึงของเราเอง
  • ทดสอบกับข้อมูล dev ก่อน SQL ที่ AI สร้างมั่นใจแต่ผิดแบบแนบเนียนบ่อย (join fan-out, การจัดการ NULL, ตรรกะเขตเวลา)
  • ใช้ Cowork สำหรับโค้ดที่แตะไฟล์ของเรา เมื่อสคริปต์ต้องอ่านข้อมูลโครงการจริง ให้เขียนและรันใน Claude Cowork กับโฟลเดอร์งาน — ไม่ใช่เครื่องมือเขียนโค้ดภายนอก

เศรษฐศาสตร์โทเคน

การส่งงานสร้างคิวรีให้เอเจนต์ฟรีหรือต้นทุนต่ำ ประหยัดความจุ Claude สองต่อ: ตัวการสร้างเอง และรอบการแก้ไข (คิวรีส่วนใหญ่ต้องปรับ 2–3 รอบ) เก็บ Claude ไว้สำหรับคิวรีที่ต้องใช้บริบทสัญญางาน — "หารูปแบบที่เราคุยกันสำหรับลูกค้ารายนี้" — ซึ่งเอเจนต์ภายนอกไม่ควรเห็นอยู่แล้ว

วิเคราะห์โดยใช้บริบทสัญญางานClaude (Project)ต้องใช้ระเบียบวิธีและความลับของเรา
สร้างหรือแก้ไฟล์งานส่งมอบClaude Coworkงานไฟล์โดยตรงในโฟลเดอร์งานที่ควบคุม

เครื่องมือใหม่? ตลาด AI เปลี่ยนทุกเดือน ก่อนใช้เครื่องมือที่ไม่อยู่ในหน้านี้ ขออนุมัติจากหัวหน้าฝ่าย Innovative Technology — เราตรวจเงื่อนไขข้อมูล โฮสติ้ง และนโยบายเทรนโมเดลก่อน ระหว่างรออนุมัติ ใช้ได้เฉพาะ Tier 1

အပိုင်း ၅

ပိုကျယ်ပြန့်သော AI ကိရိယာစုံ

အခမဲ့ LLM များနှင့် အထူးပြု coding agent များက အခပေး Claude စွမ်းရည်ကို လိုအပ်သောအလုပ်အတွက် ချန်ထားနိုင်စေသည်။ အလဲအလှယ်မှာ ထိန်းချုပ်မှုဖြစ်သည် — ကိရိယာစျေးပေါလေ ထိတွေ့နိုင်သောအကြောင်းအရာ အထိခိုက်မခံရလေဖြစ်သည်။

အခမဲ့ LLM များ — DeepSeek နှင့် Kimi

DeepSeek (DeepSeek AI) နှင့် Kimi (Moonshot AI) သည် စွမ်းရည်ပြည့်အခမဲ့မော်ဒယ်များဖြစ်သည် — coding၊ စာတမ်းရှည်ကိုင်တွယ်မှုနှင့် ယေဘုယျမူကြမ်းရေးရာတွင် တော်ပြီး ယေဘုယျအလုပ်များတွင် အခပေးမော်ဒယ်များနှင့် ယှဉ်နိုင်လေ့ရှိသည်။ Tier 1 အလုပ်အတွက် ထူးချွန်သော တိုကင်ချွေတာရေးကိရိယာများဖြစ်သည်။

တင်းကြပ်သောကန့်သတ်ချက် — Tier 1 သာ။ ဤဝန်ဆောင်မှုများသည် ကျွန်ုပ်တို့မထိန်းချုပ်နိုင်သောနေရာတွင် လည်ပတ်နေပြီး ဒေတာကိုင်တွယ်မှုစည်းကမ်းများကို ကျွန်ုပ်တို့ အတင်းအကျပ်မလုပ်နိုင်ပါ — ENVISAGE လုပ်ကိုင်သောနေရာနှင့် ကျွန်ုပ်တို့သုတေသနပြုသောအရာများကြောင့် ကျွန်ုပ်တို့၏အကြောင်းအရာသည် ထိုအခြေအနေတွင် ထူးခြားစွာအထိခိုက်မခံပါ။ အတွင်းပိုင်းအရာ၊ ဖောက်သည်ဆိုင်ရာအရာ၊ ကျွန်ုပ်တို့သုတေသနမှနာမည်များ — ဘယ်တော့မှ မထည့်ရ။ ပို့သမျှအားလုံး သိမ်းဆည်းခံရပြီး ဖတ်နိုင်သည်ဟု ယူဆပါ။

ကောင်းသောအသုံးပြုမှုများ (Tier 1)

  • Open-source စုဆောင်းမှုစိစစ်စဉ် ထုတ်ဝေပြီးသတင်းနှင့် အများပြည်သူအစီရင်ခံစာများ အကျဉ်းချုပ်ခြင်း
  • ပထမအကြိမ် ယေဘုယျမူကြမ်း — ပုံသေဖော်ပြချက်များ၊ အစည်းအဝေးအစီအစဉ်များ၊ ယေဘုယျဆက်သွယ်ရေးပုံစံများ
  • ကျွန်ုပ်တို့၏အကြောင်းအရာတွင် အသုံးချရန် Claude တိုကင်မသုံးခင် အများပြည်သူသဘောတရား၊ စာရင်းအင်းနည်းလမ်း သို့မဟုတ် library တစ်ခုကို ရှင်းပြခိုင်းခြင်း
  • ယေဘုယျကုဒ်အပိုင်းအစများနှင့် debug အကူအညီ (schema နာမည်မပါ၊ အတွင်းပိုင်းကုဒ်မပါ)
  • ENVISAGE အသေးစိတ်မပါသော ဖွဲ့စည်းပုံနှင့် checklist စိတ်ကူးထုတ်ခြင်း
ကောင်းသော လမ်းကြောင်းချမှု

အများပြည်သူဆောင်းပါး ၃၀ ကိုစိစစ်နေသော သုံးသပ်သူက Kimi ကို တစ်ပုဒ်လျှင် စာကြောင်းနှစ်ကြောင်းအကျဉ်းချုပ်ခိုင်းပြီး သက်ဆိုင်သော ၆ ပုဒ်ကိုရွေးကာ ၎င်းတို့ကိုသာ Claude ထဲသို့ စာချုပ်နှင့်ဆက်စပ်ခွဲခြမ်းစိတ်ဖြာရန် ယူသွားသည်။ အခမဲ့ကိရိယာက အစုလိုက်ဖတ်သည်; Claude က ဆင်ခြင်သည်။

ဘယ်တော့မှ မလုပ်ရ

"စကားလုံးချောအောင်လုပ်ရုံ" ဟုဆိုကာ ဖောက်သည်အကဲဖြတ်မူကြမ်းကို DeepSeek ထဲကူးထည့်ခြင်း။ ထိုစကားလုံးများသည် Tier 3 အကြောင်းအရာနှင့် တွဲနေသည် — ထိုမူကြမ်းသည် ယခု ကျွန်ုပ်တို့မထိန်းချုပ်နိုင်သော အခြေခံအဆောက်အအုံပေါ်တွင် ရှိနေပြီဖြစ်သည်။

Coding Agent များ — သာမန်စကားမှ SQL နှင့် အခြားများသို့

သာမန်စကားမှ SQL ထုတ်ပေးသော agent များ (Pi ပုံစံ NL2SQL ကိရိယာများ၊ SQL Agent CLI နှင့် အလားတူများ) နှင့် coding အကူများသည် ရိုးရိုးအင်္ဂလိပ်တောင်းဆိုချက်များကို အလုပ်လုပ်သော query နှင့် script များအဖြစ် ပြောင်းပေးသည်။ Python၊ PostgreSQL၊ Elasticsearch နှင့် Snowflake ပေါ်လည်ပတ်သောအဖွဲ့အတွက် SQL မရေးတတ်သောသုံးသပ်သူများသည် developer ကိုစောင့်စရာမလိုဘဲ အဖြေရနိုင်သည်။

လုံခြုံသောပုံစံ — schema ထည့်၊ query ထုတ်

Coding agent သည် သင်၏ query ရေးရန် သင်၏ဒေတာ ဘယ်တော့မှမလိုပါ — ဒေတာ၏ပုံသဏ္ဌာန်ကိုသာ လိုသည်။ ဇယားနှင့်ကော်လံများကို ဖော်ပြပါ; ထုတ်ပေးသော query ကို ကျွန်ုပ်တို့၏ပတ်ဝန်းကျင်အတွင်း ကိုယ်တိုင် run ပါ။

ကျွန်ုပ်တို့၏ PostgreSQL ဒေတာဘေ့စ်တွင် —
  entities(entity_id, name_hash, type, country, first_seen)
  relationships(rel_id, entity_a, entity_b, rel_type, confidence)
  mentions(mention_id, entity_id, source_id, published_date)

နောက်ဆုံးရက် ၉၀ အတွင်း ရင်းမြစ် ၃ ခု+ တွင် ဖော်ပြခံရပြီး
'ownership' ဆက်နွယ်မှု အနည်းဆုံးတစ်ခုရှိသော entity များကို
ဖော်ပြမှုအရေအတွက်အလိုက်စီ၍ ပြန်ပေးသော query ရေးပါ။

Prompt ထဲမှာ ဘာပါသလဲ သတိပြုပါ — ကော်လံနာမည်များနှင့် ရည်ရွယ်ချက်သာ — ဒေတာအတန်းမပါ၊ နာမည်အစစ်မပါ၊ စကားဝှက်မပါ။ Agent က SQL ပြန်ပေးသည်; သင်ပြန်စစ်ပြီး pipeline အတွက်ဆိုလျှင် ပြန်လှန်စစ်ဆေးကာ Sapphire အတွင်းမှာ run ပါ။

Coding agent စည်းမျဉ်းများ

  • Schema ဖွဲ့စည်းပုံသာ။ ဇယားနှင့်ကော်လံနာမည်များသည် Tier 2 — ခွင့်ပြုကိရိယာများအတွက် ရသည်။ ဒေတာအတန်းများသည် ထိုဒေတာ၏ Tier အတိုင်းဖြစ်ပြီး များသောအားဖြင့် Tier 3 ဖြစ်သည်။
  • စကားဝှက်များ ဘယ်တော့မှ မထည့်ရ — connection string များလည်း မထည့်ရ၊ ပြင်ပ agent ကို production ဒေတာဘေ့စ်နှင့် တိုက်ရိုက်မချိတ်ရ။ ထုတ်ပေးသော query များသည် ကျွန်ုပ်တို့၏ကိုယ်ပိုင်အသုံးပြုခွင့်ထိန်းချုပ်မှုများမှတစ်ဆင့် run ရမည်။
  • Dev ဒေတာဖြင့် အရင်စမ်းပါ။ AI ထုတ် SQL သည် ယုံကြည်မှုပြည့်သော်လည်း သိမ်မွေ့စွာမှားလေ့ရှိသည် (join fan-out၊ NULL ကိုင်တွယ်မှု၊ timezone logic)။
  • ကျွန်ုပ်တို့ဖိုင်များကိုထိသောကုဒ်အတွက် Cowork သုံးပါ။ Script က ပရောဂျက်ဒေတာအစစ်ဖတ်ရမည်ဆိုလျှင် ပြင်ပ coding ကိရိယာတွင်မဟုတ်ဘဲ Claude Cowork တွင် အလုပ်ဖိုင်တွဲနှင့် ရေး၍ run ပါ။

တိုကင် စီးပွားရေး

Query ထုတ်လုပ်မှုကို အခမဲ့ သို့မဟုတ် စျေးသက်သာသော coding agent သို့ လမ်းကြောင်းချခြင်းက Claude စွမ်းရည်ကို နှစ်ဆချွေတာသည် — ထုတ်လုပ်မှုကိုယ်တိုင်နှင့် ပြင်ဆင်မှုသံသရာ (query အများစုသည် ၂–၃ ကြိမ် ပြင်ရသည်)။ Claude ကို စာချုပ်အကြောင်းအရာလိုသော query များ — "ဤဖောက်သည်အတွက် ကျွန်ုပ်တို့ဆွေးနွေးခဲ့သောပုံစံကို သတ်မှတ်ပါ" — အတွက်သာ ချန်ထားပါ။ ထိုအရာများကို ပြင်ပ agent က မြင်ခွင့်မရှိသင့်ပါ။

လမ်းကြောင်းချ အမြန်ဇယား

အလုပ်ပထမရွေးချယ်မှုအကြောင်းရင်း
အများပြည်သူဆောင်းပါးများ အစုလိုက်အကျဉ်းချုပ်ခြင်းDeepSeek / KimiTier 1၊ ပမာဏများ — အခမဲ့ကိရိယာများက တိုကင်ဝန်ကို ခံပေးသည်
စာချုပ်အကြောင်းအရာသုံး ခွဲခြမ်းစိတ်ဖြာမှုClaude (Project)ကျွန်ုပ်တို့၏နည်းစနစ်နှင့် လျှို့ဝှက်မှုလိုသည်
ပေးအပ်စာတမ်းဖိုင်များ တည်ဆောက်/ပြင်ဆင်ခြင်းClaude Coworkထိန်းချုပ်ထားသောအလုပ်ဖိုင်တွဲတွင် တိုက်ရိုက်ဖိုင်အလုပ်
သာမန်စကား → pipeline အတွက် SQLNL2SQL coding agentSchema သာပါသော prompt; စျေးသက်သာသောပြင်ဆင်မှု
အချက်၊ ကိန်းဂဏန်း သို့မဟုတ် ဘာသာပြန် စစ်ဆေးခြင်းGemini / ChatGPT (အခမဲ့)တိုပြီး အကြောင်းအရာမလိုသော စစ်ဆေးရေး prompt များ
ယေဘုယျကုဒ် ရှင်းပြ/debug ခြင်းDeepSeek / KimiCoding စွမ်းရည်ကောင်း၊ ကုန်ကျစရိတ်မရှိ၊ Tier 1 အကြောင်းအရာ
လူ၊ ဖောက်သည် သို့မဟုတ် ရင်းမြစ်နာမည်ပါသမျှClaude သာ — အမည်ဖျောက်၍Tier 3 သည် ခွင့်ပြုကိရိယာများမှ ဘယ်တော့မှမထွက်ရ

ကိရိယာအသစ်လား? AI စျေးကွက်သည် လစဉ်ပြောင်းလဲသည်။ ဤစာမျက်နှာတွင်မပါသော ကိရိယာတစ်ခုခုမသုံးခင် Innovative Technology ခေါင်းဆောင်နှင့် ညှိနှိုင်းပါ — ၎င်း၏ဒေတာစည်းကမ်း၊ hosting နှင့် training မူဝါဒကို အရင်စစ်သည်။ မခွင့်ပြုသေးသမျှ Tier 1 အဖြစ်သာ ဆက်ဆံပါ။
